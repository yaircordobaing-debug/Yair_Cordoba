# Portafolio 2.0 - [Nombre] 🚀

Un portafolio web integral, modular y de alto rendimiento diseñado estructuralmente sin frameworks pesados. Su enfoque principal es demostrar un dominio absoluto sobre la ingeniería de software fundacional, la resolución de problemas algorítmicos y crear una marca personal de alto impacto.

## 🌟 Características Principales

*   **Arquitectura de 19 Páginas Aisladas:** Separación de preocupaciones a nivel de rutas, facilitando la escalabilidad, accesibilidad y el SEO orgánico.
*   **Web Components Nativos en Vanilla JS:** Cero repetición de código para la barra de navegación `<nav-bar>` y el pie de página `<footer-bar>`. Mantenibilidad a largo plazo asegurada.
*   **Motor Visual Glassmorphism:** Sistema puramente en CSS nativo, unificando el efecto de cristalización de fondos para una experiencia UI/UX inmersiva.
*   **Animaciones Asíncronas Óptimas:** Animaciones `fade-in-up` fluidas autogestionadas por `IntersectionObserver` genérico, respetando a 60 fps el Main Thread móvil.

## 📄 Detalle Arquitectónico por Página

Para mantener una separación limpia de conceptos, a continuación se detalla exactamente la información contenida en cada página:

1. **`index.html` (Home/Landing):** El punto de entrada central. Contiene el Hero impactante, el resumen rápido de qué aportas, tu "Frase Personal Científica" y el "Llamado a la Acción (CTA)" principal orientado a invitar a trabajar juntos.
2. **`about.html` (Sobre mí):** Una inmersión narrativa. Contiene el Pitch profesional completo, la historia cronológica de "Mi Camino en Ingeniería" (los altibajos aprendiendo desarrollo) y un panel ejecutivo sobre enfoques universitarios.
3. **`projects.html` (Catálogo de Proyectos):** Galería principal mostrando arquitecturas reales (Frontend React, Backend Logística, Motores Gráficos Innovadores). También adelanta las "Ideas Futuras" por investigar.
4. **`project-detail.html` (Análisis Profundo):** Plantilla reutilizable detallando metodológicamente un desarrollo: "La Barrera Técnica" (El cuello de botella), "Arquitectura Planteada" y "El Impacto Cifrado" logrado (Tiempos de latencia recuperados).
5. **`case-studies.html` (Proceso Documental):** Explicación de 4 pasos sobre cómo diagnosticas problemas antes de programar para asegurar arquitecturas elegantes y útiles.
6. **`stack.html` (Caja de Herramientas):** Un panel altamente visual de las tecnologías dominadas (Node, React, Infraestructura) y una sección dedicada estrictamente a listar **qué Valor Aportado** te dieron los Certificados formales que ganaste.
7. **`process.html` (Cómo construyo software):** Gráfica vertical de las fases cardinales (Requerimientos Crudos -> Diseño Interactivo -> Desarrollo Desacoplado -> CI/CD Control OQA).
8. **`challenges.html` (Retos Resueltos):** Relatos precisos sobre cómo mitigaste bugs horribles (por ejemplo, consultas superpuestas SQL N+1) utilizando el poder asimétrico del método científico, diagramado.
9. **`engineering.html` (Investigación Académica):** Detalles sobre tu participación en el Semillero SITEM, desarrollo IoT y publicaciones académicas formales (Revista Poligran).
10. **`learning.html` (Aprendizaje Continuo):** Exposición de resiliencia frente al constante avance computacional mediante la "Teoría Abstracta del 1%".
11. **`vision.html` (El Futuro):** Visión macro sobre tu destino ambiciosamente liderando equipos en la integración algorítmica IA de ultra-bajo nivel.
12. **`impact.html` (Impacto Social):** Ensayo contundente que declara el deber cívico de tu ingeniería enfocada en inclusión visual y robustez en la sociedad tecnológica.
13. **`experience.html` (Trabajo en Equipo):** Fomento claro de porqué la excelente interfaz se da gracias a *soft skills* impecables, comunicación en Git profunda y debates amables.
14. **`blog.html` (Central de Artículos):** Índice visual para agrupar notas sobre cómo la IA se expande o debates de reactividad vs mutabilidad estructurada.
15. **`post.html` (Artículo Individual):** Contenedor asimétrico de interfaz blanca ideal, que solo favorece sumergir a un lector sin molestar u obstruir flujos.
16. **`books.html` (Biblioteca Formativa):** Lecturas base como "Hábitos Atómicos" y manuales lógicos pragmáticos en grilla.
17. **`hobbies.html` (Pasiones/Playlist):** Humanización estricta. ¿Qué escuchas cuando codificas horas y horas? (*Deep Focus*). ¿Por qué vas al gym? (Cortisol reiniciado).
18. **`manifesto.html` (El Manifiesto Tech):** Un documento gigantesco y poético incrustado que dictamina el norte sagrado ético y operacional de tu labor en la carrera elegida.
19. **`contact.html` (Interacción / Conversión):** Un formulario de captación estilizado de alto impacto (UI Focus Glass), con links unificados de LinkedIn/Github listos para captar ofertas B2B en segundos.

## 🚀 Cómo Ejecutar

El proyecto es **100% estático**.
1. Clona/Descarga la carpeta.
2. Abre `index.html` usando extensiones tipo **Live Server** de *VS Code* (Obligatorio para cargar sin bloqueos CORS los Web Components Locales `js/components.js`).
